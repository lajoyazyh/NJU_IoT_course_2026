一、实验目的  
智能家居技术通过物联网（ IoT ）设备将日常家电连接到网络，使得用户能够远程控制和管理联网设备。
Cisco Packet Tracer 是一个网络虚拟工具，能够模拟家庭网络的智能设备和网络通信等情况。本实验将
使用 Cisco Packet Tracer 模拟一个智能家居环境，让学生了解物联网的日常应用，激发学生学习兴趣。
同时，传授学生实验方法。鼓励学生自己设计并实现各类应用场景。
二、实验内容及目标  
1. 在 Cisco Packet Tracer 中配置⼀个基本的智能家居⽹络。  
2. 实现对智能家电设备的远程控制（如摄像头、智能门窗等）。
3. 学习如何在局域⽹内配置⽆线⽹络，并通过笔记本或智能⼿机控制智能设备。
三、实验步骤  
实验环境  
工具： Cisco Packet Tracer 
网络拓扑：家庭⽹络模型，包括：  
无线路由器：为智能设备提供⽆线连接。
智能家电设备：智能摄像头、智能门窗等可远程控制的家具。
 控制设备：智能⼿机、平板等。
实验一、创建智能家居网络 ( 必做 )  
步骤一：添加网络设备。从 Network Devices > Wireless Devices 中拖放⽆线路由器（如 Generic 
WRT300N  Wireless Router ）到⼯作区。从 End Devices 中拖放⼀部智能⼿机和平板到⼯作区，这些设
备⽤于远程控制家电。
步骤⼆：添加智能家电设备。从 Home > IoT Devices 中拖放以下智能家电设备到⼯作区，确保这些设备
通过⽆线  连接与⽆线路由器相连。
步骤三：  配置无线网络：配置路由器。  
步骤四：连接智能设备：连接控制设备（智能⼿机和平板），确保智能⼿机和笔记本电脑通过⽆线⽹络
连接到⽆线路由器，检查⽹络连接是否正常。将智能设备连接到家庭 Wi-Fi ⽹络。双击每个设备，进⼊⽹
络设置，确保设备成功连接到路由器并获得 IP 地址。
 步骤五：配置智能设备控制：如在智能摄像头设备设置中，选择 On/Off 功能，允许设备通过⽹络进⾏远
程控制。
 步骤六：测试远程控制：通过笔记本电脑或智能⼿机的浏览器，输⼊路由器 IP 地址，进⼊智能家居控制
⾯板。远程控制智能设备的反应速度和控制效果。记录每次控制操作是否成功，观察设备的反馈状态。
 
实验二、模拟防盗系统 ( 必做 )  
场景设置：为了防止偷盗，设计一个监视场景。要求：当有人触发报警装置时，关闭门防止窃贼逃跑，
同时打开网络摄像头进行拍摄。
步骤一：添加无线路由器和服务器。
步骤二：添加触发装置 (Trip sensor) 、网络摄像头 (Webcam) 和智能门 (Door) 。
步骤三：配置无线网络，使用服务器管理设备。
步骤四：根据场景要求设置对应的条件。
步骤五：模拟测试。
 
实验三、模拟养殖场 ( 选做 )  
场景设置：模拟恒温养殖，要求
1. 养殖场的温度在 20 ℃ -30 ℃之间 ( 超过 30 摄氏度，自动打开空调；低于 20 摄氏度，自动打开加热
器。 20-30 摄氏度均不打开。空调和加热器不能同时打开。 )
2. 内置摄像头以供用户通过手机查看。若用户打开摄像头，必须有灯光 ( 台灯或者路灯均可 ) 同步打
开。
