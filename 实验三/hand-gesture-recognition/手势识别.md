# 手势识别实验指南

---

## 一、实验概述

### 1.1 实验背景

手势识别是物联网应用软件开发中的典型智能感知任务。摄像头或图片作为感知输入，边缘端程序完成手部检测、关键点提取、手势分类和结果输出，最后可以进一步驱动灯光、门禁、屏幕控制、智能家居或人机交互系统。

本实验基于开源项目 `hand-gesture-recognition-using-mediapipe`，使用 MediaPipe 提取 21 个手部关键点，再使用 TensorFlow Lite 模型识别手势。原项目默认使用摄像头实时识别；考虑到部分电脑没有摄像头，本实验已增加图片输入模式，学生可以直接对图片进行识别并保存标注结果。

### 1.2 实验目标

完成本实验后，学生应能够：

- 理解“图像采集 -> 手部检测 -> 关键点提取 -> 特征归一化 -> 手势分类 -> 应用输出”的完整流程。
- 掌握 Python 视觉项目的依赖安装、运行和常见错误排查方法。
- 能够使用命令行对单张图片进行手势识别并生成结果图。
- 能够修改 OpenCV 标注代码，解决文字被裁切、不可见等实际工程问题。
- 能够设计一个简单的物联网手势控制应用逻辑。
- 能够完成若干小型拓展实验，使项目从“能运行”变成“能分析、能改造、能应用”。

### 1.3 项目功能介绍

本项目包含两类识别功能：

| 类型 | 功能 | 当前标签 |
|---|---|---|
| 静态手势识别 | 根据单帧手部关键点识别手势 | `Open`、`Close`、`Pointer`、`OK` |
| 动态手势识别 | 根据食指指尖历史轨迹识别动作 | `Stop`、`Clockwise`、`Counter Clockwise`、`Move` |

本次课堂重点使用静态手势识别。动态手势识别需要连续视频帧或摄像头输入，可作为拓展讨论。

### 1.4 物联网应用场景

| 场景 | 应用描述 |
|---|---|
| 智能家居 | 张开手掌开灯，握拳关灯，OK 手势确认操作 |
| 智能展厅 | 游客通过手势控制屏幕翻页、选择展品 |
| 无接触设备控制 | 在医院、工厂等场景下减少实体按钮接触 |
| 边缘 AI 网关 | 摄像头采集画面，边缘设备本地完成识别，只上传识别结果 |
| 辅助教学 | 识别学生举手、确认、指向等课堂动作 |

## 四、基础实验：运行图片手势识别

### 4.1 原项目摄像头模式

原项目默认使用摄像头：

```bash
python app.py
```

将待识别图片放到项目目录，例如：

| 参数 | 说明 |
|---|---|
| `--image` | 输入图片路径 |
| `--output` | 输出结果图片路径 |
| `--no_display` | 不弹出 OpenCV 窗口，只保存图片 |

```text

Saved result: result.jpg
```

打开 `result.jpg`，应能看到：

- 手部关键点。
- 手部骨架连接线。
- 手部外接框。
- 手势识别标签，例如 `Right:Open` 或 `Left:Open`。

如果没有识别到手，会输出：

```text
No hand detected.
Saved result: result.jpg
```

## 五、核心原理讲解

### 5.1 系统处理流程

```text
输入图片/摄像头
      ↓
OpenCV 读取图像
      ↓
MediaPipe Hands 检测手部
      ↓
提取 21 个手部关键点
      ↓
关键点坐标相对化和归一化
      ↓
TFLite 静态手势分类器
      ↓
OpenCV 绘制骨架、边界框、标签
      ↓
保存 result.jpg 或显示窗口
```

### 5.2 MediaPipe 手部关键点

MediaPipe Hands 会输出 21 个手部关键点，包括手腕、拇指、食指、中指、无名指和小指的关节点。

项目中的关键函数：

| 函数 | 作用 |
|---|---|
| `calc_landmark_list()` | 将 MediaPipe 的归一化坐标转换为图片像素坐标 |
| `pre_process_landmark()` | 将绝对坐标转换为相对坐标，并做归一化 |
| `KeyPointClassifier()` | 加载 TFLite 静态手势模型并推理 |
| `draw_landmarks()` | 绘制手部骨架和关键点 |
| `draw_info_text()` | 绘制识别标签 |

### 5.3 为什么要做坐标归一化

同一个手势可能出现在图片的不同位置、不同大小。如果直接使用像素坐标，模型会受到图片尺寸和手掌位置影响。

项目采用的处理方式：

1. 以第 0 个点，即手腕位置，作为基准点。
2. 所有关键点坐标减去手腕坐标，变成相对坐标。
3. 找到最大绝对值，用它对所有坐标进行缩放。
4. 得到长度固定的 42 维输入特征，即 21 个点的 x/y 坐标。

这使模型更关注“手指之间的相对形状”，而不是手在图片中的绝对位置。

### 5.4 静态手势模型

静态手势分类器位于：

```text

```

标签位于：

```text

```

```text

Close
```

## 六、调试记录与常见问题

### 6.1 问题 1：缺少 MediaPipe

报错：

```text
ModuleNotFoundError: No module named 'mediapipe'
```

原因：当前 Python 环境没有安装 `mediapipe`。

解决：

```bash
pip install mediapipe==0.10.21
```

### 6.2 问题 2：新版 MediaPipe 不支持旧接口

报错：

```text
AttributeError: module 'mediapipe' has no attribute 'solutions'
```

原因：安装了较新的 `mediapipe 0.10.35`，该版本主入口只暴露了 `tasks`，原项目使用的 `mp.solutions.hands` 接口不可用。

解决：降级到兼容旧接口的版本。

```bash
pip install mediapipe==0.10.21
```

验证：

```bash
python -c "import mediapipe as mp; print(mp.__version__); print(hasattr(mp, 'solutions'))"
```

应输出：

```text
0.10.21
True
```

### 6.3 问题 3：TensorFlow 与 protobuf 冲突

报错示例：

```text
ImportError: cannot import name 'runtime_version' from 'google.protobuf'
```

原因：`tensorflow 2.21` 需要较新的 `protobuf`，而 `mediapipe 0.10.21` 依赖较旧的 `protobuf`，二者冲突。

解决：使用与 `mediapipe 0.10.21` 更兼容的 TensorFlow 版本。

```bash
pip install tensorflow==2.18.1
```

### 6.4 问题 4：没有摄像头

现象：运行原命令 `python app.py` 后无法采集图像，或者窗口没有有效画面。

解决：使用本实验改造后的图片输入模式。

```bash
python app.py --image hand.jpg --output result.jpg --no_display
```

### 6.5 问题 5：结果图片中标签看不见

原因：原代码把文字标签固定画在手部边界框上方。如果手指接近图片上边缘，标签会被画到图片外。

修改思路：

- 先判断边界框上方是否有足够空间。
- 如果上方空间不足，就把标签画到边界框下方。
- 使用 `cv.getTextSize()` 根据文字宽度动态计算黑色背景框。

涉及函数：

```text
draw_info_text()
```


学生大约几分钟就能完成，学习深度不足。建议将课程重点放在：

- 为什么原项目依赖会冲突。
- 为什么摄像头程序要改造成图片程序。
- 为什么标签会被画到图片外。
- 检测阈值如何影响识别结果。
- 如何把识别结果转化为物联网控制动作。

这样学生不仅“跑通了代码”，还经历了真实工程中的环境适配、代码改造、问题排查和应用设计。

### 10.2 推荐课堂最小闭环

如果课堂时间紧，建议至少完成：

1. 图片模式运行。
2. 标注位置修复。
3. 阈值敏感性实验。
4. IoT 控制规则设计。

这四项能覆盖环境、视觉识别、代码修改和应用落地。
