# 基于多人人脸检测与表情识别的课堂状态分析系统 — 开发计划

## 一、现状分析

### 已有可复用资源

| 来源 | 可复用内容 | 说明 |
|------|-----------|------|
| **实验二** | CNN3 表情识别模型 + 训练权重 | Keras/TensorFlow，支持 7 类表情（anger/disgust/fear/happy/sad/surprised/neutral），准确率较好 |
| **实验二** | 人脸检测逻辑 | 支持 Haar Cascade + BlazeFace 双检测器，已有多人脸检测能力 |
| **实验二** | `generate_faces()` 预处理 | 人脸增广提升识别准确率 |
| **实验二** | `cv2_img_add_text()` | 中文标注工具函数 |
| **实验三** | Streamlit 界面经验 | 实验三已使用 Streamlit，技术栈一致 |

### 实验二代码的不足（需要改进）

1. `predict_expression()` 只返回**第一个人**的表情结果（`return emotions[0]`），而大作业需要**所有**人的结果
2. 界面使用 PyQt5，大作业推荐 Streamlit/Gradio
3. 缺少表情统计、课堂状态判断、CSV 记录等功能

## 二、技术选型

| 模块 | 技术方案 | 理由 |
|------|---------|------|
| **人脸检测** | MediaPipe Face Detection（优先）+ OpenCV Haar Cascade（回退） | 实验二已验证，MediaPipe 精度更好 |

| **图表展示** | Streamlit 内置图表 + Altair/Matplotlib | 表情分布柱状图、历史趋势图 |

## 四、模块详细设计

### 4.1 `face_detector.py` — 人脸检测模块

```
函数:
  detect_faces(image) → List[bbox]  # 返回所有检测到的人脸框坐标
  draw_bboxes(image, faces) → image  # 在原图上绘制人脸框
```

- 优先使用 MediaPipe Face Detection（精度高）
- 检测失败时回退到 OpenCV Haar Cascade
- 返回格式：`[(x1, y1, x2, y2, confidence), ...]`

### 4.2 `expression_recognizer.py` — 表情识别模块

```
类: ExpressionRecognizer
  __init__(model_path)        # 加载 CNN3 模型权重
  predict(face_image) → (label, confidence, probs)  # 单个人脸识别
  recognize_all(image, faces) → List[result]  # 批量识别
```

- 复用实验二的 CNN3 模型结构
- 复用 `generate_faces()` 增广策略
- 返回中英文标签 + 置信度 + 7 类概率分布

### 4.3 `analyzer.py` — 统计与分析模块

```
函数:
  calculate_statistics(results) → dict  # 计算表情分布
  judge_classroom_state(stats) → str    # 课堂状态判断
```

**课堂状态判断规则（按优先级）：**
1. 检测人数 = 0 → "未检测到学生"
2. Happy + Neutral 占比 ≥ 70% → "课堂状态良好 😊"
3. Sad + Angry 占比 ≥ 40% → "课堂状态较低落，需要关注 ⚠️"
4. Neutral 占比最高 → "课堂状态平稳 😐"
5. Surprise 占比较高 → "课堂注意力波动较大 😮"
6. 其他 → "课堂状态一般"

## 七、关键设计决策

1. **为什么用 Streamlit 而不是 PyQt5？** 大作业明确推荐 Streamlit/Gradio，实验三已验证可用，开发效率高，内置图表和文件上传组件，无需写复杂前端代码。

2. **为什么保留 Haar Cascade 回退？** 实验二代码中 MediaPipe 有时在静态图上漏检，保留 Haar 回退可提升稳定性。

3. **是否需要重新训练模型？** 不需要。实验二的 CNN3 模型已用 FER2013 训练好，覆盖 7 类表情，直接复用即可。

4. **延展任务（时序分析）如何处理？** 单独一个扩展模块 `temporal_analyzer.py`，实现帧采样、滑动窗口、三级预警和时序可视化，不影响基础功能。
